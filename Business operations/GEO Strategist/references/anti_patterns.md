## Anti-Pattern 8: Ignoring Mobile Users

### âŒ Bad Example
```
- Tiny text requiring zoom
- Buttons too small to tap
- Horizontal scrolling required
- Pop-ups covering content
- Desktop-only navigation
```

**Why it fails:**
- 60%+ of traffic is mobile
- Frustrating user experience
- Google penalizes mobile-unfriendly sites
- Lost conversions

**What happens:**
- High mobile bounce rate
- Lower search rankings
- Negative brand perception
- Lost sales

### âœ… Good Alternative
```
- Large, readable text (minimum 16px)
- Big tap targets (minimum 44x44px)
- Sticky CTA button always visible
- Fast loading (compressed images)
- One-column layout on mobile
```

**Why it works:**
- Matches how users actually browse
- Reduces friction
- Google rewards mobile optimization
- Higher conversion rates

## Summary: The Core Mistake

**The fundamental error underlying all anti-patterns:**

> Focusing on what YOU want to say instead of what YOUR BUYER needs to know.

**The fix:**

> Start every page by asking: "What is stopping this person from requesting a quote right now?"

Then structure the entire page to remove those specific barriers.

---

## Quick Audit Checklist

Run any page through these questions:

- [ ] Does the H1 address a specific pain point or desired outcome?
- [ ] Is pricing transparent (or reason for quote explained)?
- [ ] Are features linked to benefits?
- [ ] Is there a clear primary CTA above the fold?
- [ ] Do images show real products in real settings?
- [ ] Is the page scannable (bullets, short paragraphs)?
- [ ] Are there trust signals (reviews, certs, guarantees)?
- [ ] Is it mobile-friendly?
- [ ] Are keywords used naturally?
- [ ] Is urgency real (not fake scarcity)?
- [ ] Are common objections addressed?

If you answer "no" to any of these, you've found an anti-pattern to fix.
